使用 Mac OS X 的 WebDAV 上載檔案至資源

WebDAV 是指「網路分散式授權與版本控制」。它是一種 HTTP 的擴展協議，透過該協議使用者可協同編輯和管理遠端網路伺服器上的檔案。

如果您擁有適當的權限，那麼就可使用 WebDAV 上載並存取網站資源中的檔案。

考慮下列事項：

  • 設定 WebDAV 前，目標資源工具中必需至少含有一個物件（即檔案或資料夾）。如需説明，請參考資源。

  • 直接從應用程式（例如 Word 或 Excel）將檔案儲存至網站，這種操作並不可靠。為了獲得最佳效果，請先將檔案儲存到您的電腦中，然後上載至網站。

註釋：不支援含有某些字元的檔案名。如需檢視不支援字元清單，請參考不支援的字元。

Mac OS X 10.4 (Tiger) 和 10.5 (Leopard)

如需使用 WebDAV 上載和存取 Mac OS X 10.4 (Tiger)及更高版本中的檔：

  1. 在 Finder 中按 Cmd-k。（或者從轉到功能表選擇連接到伺服器。）

  2. 在「伺服器地址：」欄位中輸入指向目標資源工具的 URL。

    與網站管理員聯絡瞭解 URL 格式；該格式中可能包含網站的網站 ID。

註釋：如需瞭解如何尋找網站 ID 的說明，請參考尋找網站的網站 ID

您現在可在電腦和您剛才連接的網站之間拖放資料夾。

或者您也可以按一下伺服器地址右邊的+（加號），將伺服器新增到您的伺服器收藏夾中。

登入網站時，可選擇記住密碼。如果選擇記住密碼，那麼當您登入個人資源或您曾經存取的網站資源時，將不再提示您輸入密碼。除非您的電腦受到保護不會被他人使用，否則請勿選擇該選項。

註釋：當透過這一方法使用 WebDAV 存取資源時，您會發現自動建立 .DS_store 檔案。蘋果電腦的知識庫中提供了如何去除這種現象的說明。

Mac OS X 10.3 (Panther)及更低版本

使用 WebDAV 在 Mac OS X 10.3 (Panther)及更低版本中上載和存取檔需要使用 Goliath 第三方使用者端軟體。

註釋：Goliath 存在一些已知的性能問題，無法始終正常工作。由於這一情況，如果您遇到麼問題，只能獲得有限的支持資源。

您可以從下列地址下載免費 Goliath：

http://www.webdav.org/goliath/installs/Goliath1_0_1.dmg.gz

檔案下載完成後，按兩下檔案解壓縮。然後按兩下生成的 .dmg 檔案。現在將出現名為 Goliath 1.0.1 的磁碟映射檔。遵循下列步驟進行操作：

  1. 將磁碟映射檔中包含的 Goliath 資料夾拖放至硬碟。

  2. 開啟 Goliath 應用程式。

  3. 在「伺服器地址：」欄位中輸入指向目標資源工具的 URL。

    與網站管理員聯絡瞭解 URL 格式；該格式中可能包含網站的網站 ID。

註釋：如需瞭解如何尋找網站 ID 的說明，請參考尋找網站的網站 ID

第一次嘗試連接時，可能會提示您接受網站證書。如果出現該提示，按一下確定。

您現在可在電腦和您剛才連接的網站之間拖放資料夾。
