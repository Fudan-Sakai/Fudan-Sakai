I. Introduction

The Sakai gradebook is a collaborative effort to produce a simple, open source tool for calculating, storing and disseminating grade information online. Design and development is being led by MIT and UC Berkeley with key support from the other Sakai core institutions. Plans for further development of the Sakai gradebook are outlined in Section VII.

II. Summary

A. The Sakai 2.1 gradebook will enable instructors to:

  1. Autocalculate course grades
  2. Establish and edit a mapping of the course letter grade to a 100 percent scale
  3. Add, view, edit and release to students the class assignments/assessments (straight point values only)
  4. Manually enter, view, edit and release to students their scores and grades
  5. Automatically import assignments and scores from Samigo
  6. Export scores and grades in excel, csv format
  7. Filter student lists by section
  8. Keep log of grading activity for each student's work

B. The Sakai 2.0 gradebook will enable each student to:

  1. View his/her own scores and grades, if they have been released by the instructor(s).

III. Functional Requirements

All actions described in sections I-VIII are available only to the Instructor role. The sections correspond to all nine available views in the 2.0 gradebook.

A. Gradebook Menu

  1. Link to inactive views. The Overview is the entry page.

B. Overview

  1. Sort display table by assignment name, due date, class average and total points possible
  2. View class average scores for each assignment
  3. Access the assignment details views
  4. Access the course grade view

C. Assignment Detail

  1. View total possible points for an assignment and the class average
  2. Page through all assignment summaries one by one
  3. Filter the grading table by student name or id.
  4. Set the number of students to view per page of the grading table
  5. Page through all students based on the number of students to view per page
  6. Access the assignment settings page
  7. Remove assignments
  8. Sort the grading table by student name, student ID and score
  9. Enter, save and edit assignment scores
  10. Filter student list by section
  11. Display grade change log for each student

D. Course Grade Detail

  1. View total possible points for all assignments and the cumulative class average
  2. Filter the grading table by student name or id.
  3. Set the number of students to view per page of the grading table
  4. Override autocalculated course grade
  5. Revert overridden autocalculated course grade
  6. Sort the grading table by student name, student ID, cumulative assignment point totals per student and the corresponding percentages, and Course Grade.
  7. Export student name, student ID, and Course Grades
  8. Filter student list by section
  9. Display grade change log for each student

E. Student Roster

  1. Filter the grading table by student name or id.
  2. Set the number of students to view per page of the grading table
  3. Page through all students based on the number of students to view per page
  4. View all assignments, each student's scores on all assignments, and the cumulative assignment point totals for each student.
  5. Export student name, student ID, and scores for all assignments in .csv and .xls
  6. Filter student list by section

F. Add/Edit Assignment

  1. Enter an assignment name, a required step
  2. Enter an assignment point value, a required step
  3. Choose whether to account assignment towards final grade
  4. Select an assignment due date, an optional step

G. Grade Options

  1. Choose to display assignment scores to students
  2. Choose to display the course grade to students
  3. Select the grade type for the course grade and assignment level grades in the student view: letter grades, letter grades + - (default), pass/fail
  4. Define percent ranges for letter grades or the pass/fail threshold

H. Remove assignment

  1. Select remove option

I. Student View

Each student will be able to

  1. View his or her cumulative score for all assignments and corresponding percentage
  2. View his or her course grade, if released by instructor
  3. Sort assignments by title, due date, his or her score, total possible points per assignment, his or her grade for each assignment

IV. Roles and Groups

The Sakai site and section tools will enable authorized users to:

  1. Add other instructors. The gradebook will recognize only a single instructor role. Once added to a site, all instructors will be fully authorized to view, create and edit in all areas of a gradebook.
  2. Add a Teaching Assistant. The gradebook will recoginize only a single TA role. Once added to the site and assigned one or more sections to manage, the TA will be authorized to enter grades exclusively for the students in those assigned sections.
  3. Add students. The gradebook will recognize only a single student role.

V. Wireframes/Functional Specification

A. Overview -- View 1

Wireframe of View 1

1. Table Sorting Rules

  1. Initial sort is by due date.
  2. If there is no due date, sort to the bottom of the list.
  3. The course grade always appears at the bottom of the list, regardless of the active sort.
  4. Subsorting for non-default sorts is by assignment name.
  5. The sort order is preserved for the duration of the session.

2. Assignment Title Link

  1. If the assignment was created in the gradebook, the title links to the Assignment Detail, View 2.
  2. If the assignment was created in Quiz and Test, there is text indicating that the user must go to that tool to make any changes to the assignment.
  3. The assignment name has a maximum character length of 255 and it is displayed in its entirety. Text will wrap according to the display needs of the table. There are currently no limitations on character types in the name. Internationalization requirements TBD.

3. Average Performance Values ('Avg')

  1. The average percent of points given is noted. Values are displayed as integers, and are not rounded up. (e.g. an average of 87.88 is displayed as '87')
  2. The Average points is the mean of all values >= 0 that have been entered for the assignment. Blanks are not treated as zeros for purposes of the calculation.

4. Points

  1. Points possible for this assignment
  2. If this assignment is not counted towards the final grade, the number is enclised in parenteses. This number is no counted toward the total points for the course grade.

B. Assignment Detail -- View 2

Wireframe of View 2

Grade change log popup

1. Assignment Summary

  1. The assignment title has a maximum character length of 255 and it is displayed in its entirety. There are currently no limitations on character types in the name.
  2. The average score is the mean of all values >= 0 that have been entered for the assignment. Blanks are not treated as zeros for purposes of the calculation. Values are displayed as integers, and are not rounded up. (e.g. an average of 87.88 is displayed as '87')
  3. Options link: 'Edit assignment settings' links to View 4.
  4. Options link: Remove assignment from gradebook' links to View 6.

2. Previous/Next Assignment Navigation

  1. All unsaved score values will be discarded upon Previous/Return/Overview navigation
  2. Entered search values for which find has not been activated will carry over upon Previous/Overview navigation
  3. The set of students currently being viewed will carry over upon Previous/Overview navigation, i.e., if students 21-40 are being viewed in assignment 1, those same students will be in the active view when Next is clicked.

3. New scores saved message

  1. Example confirmation message.

4. Table Sorting Rules

  1. Initial sort is by student name.
  2. Blanks or no scores sort to the top of the list
  3. Subsorting for non-default sorts is by student name.
  4. The sort order is preserved for the duration of the session.
  5. When sorting is changed, the paging resets to the first page

5. Find Rules

  1. Initial value: as in wireframe. When the focus comes to this field, the initial value disappears.
  2. If there is no match, the page reloads, there is a note where the listing table would be saying no results found for "[the search term]" The "Find" and "Assignment" values return to the initial values.
  3. The last established students per page value is retained and displayed. (Use the style specified for alert messages in the style guide.)
  4. For search values, only the last space-or-comma-separated value is treated as a substring:

    Copp
    Francis Ford Copp
    Coppola, Francis F
    ffcoppola@niebaum.edu
    ffcopp

    All return Coppola, Francis Ford

    Francis F. Coppola

    Returns an error

  5. Case is ignored.
  6. Truncation or wild card symbols are not supported.
  7. Clear resets the view to the default, except for the students per page, if that value has been changed during the session.
  8. The enter key executes Find if the search box has focus

6. View filter

  1. If the user is only authorized to administer one section, or if there is only one section in this class, this drop down menu does not appear.
  2. When the user can administer multiple sections, the dropdown menu offers an option for each section/group as well as an option for "All groups."

7. Save

  1. The Save button saves newly entered data and refreshes the view (average score is updated and 'changes saved' message added to top of page). The last executed search and students per page values are retained and displayed upon refresh.
  2. The save button is the only mechanism for saving score data. The enter key executes Save if the score boxes have focus.
  3. If newly entered data is not saved, it will be discarded upon using any of the gradebook's internal (as opposed to the browser's or Sakai framework's) navigation or find/show functionality. The gradebook supports the back button, although this is not currently true of most Sakai tools.
  4. If two or more users are simultaneously attempting to edit the same student's score, the second of the scores to be submitted will overwrite the first.
  5. If bad score values are entered, upon save an error message is generated

8. Cancel button

  1. Any new form data is discarded and the user returns to the Overview page.

9. Point Entry

  1. Values must be >= 0. Decimal values of up to 2 places are accepted, e.g. 9.75. Commas and other non-numerical characters are not accepted.
  2. Illegitimate values generate an error message: "Please enter only scores >=0 of up to 2 decimal places. "
  3. Error messages are also generated next to each of the fields with bad values: "Please enter valid grade."
  4. After values are saved, the maximum number of visible characters in text box = 3; these are right aligned. Values > 3 characters are scrollable.
  5. Scores submitted via the Tests and Quizzes tool cannot be edited in the gradebook.

10. Grade Change Log

  1. Clicking the log icon reveals the div shown in image above.
  2. Every grade change made, including the initial grade set, is listed in reverse chronological order.
  3. The date, score set, and person or agent making the change is shown
  4. A small "[x]" in the corner of the div cna be clicked to hide the div again.

C. Final Grade detail -- View 2a

This view is similar to view 2. UI components not in View 2 are noted.

Wireframe of View 2a

1. Course Summary

  1. Points: Total possible points of all assignments currently entered into the gradebook directly and/or via SAMigo.
  2. Average Score: The mean of all possible score values for all assignments. Blank scores are treated as zeros for calculation purposes.

2. Calculated Grade

  1. The calculated grade field is by default filled with the grade calculated by the gradebook (A, B . . .or A-, B+ . . . or P/NP) and the percentage the grade is based on.
  2. The grade is always calculated as the current percentage of points scored out of the total possible points of all assignments entered in the gradebook. Consequently, the grades that the instructors see are unlikely to reflect students' current performance in the class, unless or until all of the assignments have been scored.
  3. For purposes of calculating the course grade, a student's cumulative average percentage is never rounded up. For example, a student who achieves an overall percentage of 79.99 would receive a course grade of C+, if the lower threshold for a B- is 80%.

3. Grade Override

  1. Override values must be of the appropriate grade type (A, B . . . or A-, B+ . . .or P/NP) or an error is generated: "The gradebook uses the grade type [pass/not pass|simple letter grade|letter grade +/-]. Please enter only values that correspond to the grade type X."
  2. Error messages are also generated next to each of the fields with bad values: "Please enter valid grade."
  3. Saved overrides appear in the entry field as editable values.
  4. Values identical to the calculated course grade are treated as legitimate entries and do not generate an error.
  5. Deleting a saved overridden value, leaving the entry field blank and executing a save returns the course grade to the calculated value.

4. Export

  1. Files are exported in .xls and .csv format
  2. The file name format is "course_grade_[courseID]__date.*". If the class number is not available use 'course_grade_050611. Date format should follow the style guide.
  3. Exported fields and the names of the Excel column headers are Student Name, Student ID, Course Grade. The full student display names and complete assignment names are exported. Point values are chopped at 2 decimal places.
  4. Scores are listed in the assignment name column
  5. Point values are chopped at 2 decimal places.
  6. Exported data is for the course grade only
  7. All available roster data is exported. Filtering the view by student does not affect the output.
  8. The exported data is always sorted by the application default, student display name
  9. There is no navigation state change after clicking these buttons

D. Student Roster -- View 3

Wireframe of View 3

Search, view filters and paging work in this view in the same way as View 2.

1. Export buttons

  1. Files are exported in .xls and .csv format
  2. The file name format is "gradebook_[courseID]_assignment_name_date.*". If the class number is not available use 'gradebook-050611.Date format should follow the style guide.
  3. Exported fields and the names of the Excel column headers are Student Name, Student ID, [assignment name].
  4. The full student display names and complete assignment names are exported.
  5. Point values are chopped at 2 decimal places.
  6. Scores are listed in the assignment name column
  7. All available class data is exported. Filtering the view by student does not affect the output.
  8. The exported data is always sorted by the application default, student display name.
  9. There is no navigation state change after clicking these buttons

2. Point Display Values

  1. Values chop at 2 decimal points; they are not rounded up.

3. Student Display

  1. Limit display of the student name to the first 50 characters (use ellipses to indicate truncated name)
  2. If there is no roster data, the message "No students have been added to this course" appears beneath the Student and Student ID Roster table headers.

E. Add/Edit Assignment -- View 4

Wireframe of View 4

1. Title field

  1. This is a required field.
  2. The assignment title has a maximum character length of 255 and it is displayed in its entirety. There are currently no limitations on character types in the name. Internationalization requirements TBD.

2. Assignment point value field

  1. This is a required field.
  2. Value must be > 0.
  3. Decimals are permitted. Values are limted to 2 decimal places.
  4. If the entry is invalid, any values entered or selected in title or date are retained with the error message: "Please only enter numbers in the Assignment Point Value. The point value must be >0 and cannot exceed 2 decimal places."

3. Include in final grade

  1. This option is checked by default
  2. If not checked the assignemnt is not counted hen calculating the final grade.

4. Due date date-picker

  1. This field is optional.
  2. If any one date value is added (month, day or year), all must be present for a valid submission.
  3. The entered date is validated only to ensure that the values follow the pattern x/x/x.

5. Submit button

  1. For new assignments, if the submit validates, the user is returned to the overview page with a success message: "You have successfully added assignment X. Add another assignment."
  2. For edited assignments, if the submit validates, the user is returned to the assignment detail page with a success message "You have successfully updated assignment Y"
  3. The enter key executes Submit if the entry fields have focus. If the date picker has been used to enter the date, the enter key will not work until the user clicks in one of the text boxes.

6. Cancel button

  1. The form is not submitted. In the case of an added assignment, the user is returned to the Assignment Overview page; for an edited assignment the user is returned to the Assignment Detail page

F. Student Feedback Options -- View 5

Wireframe of View 5

1. Grade Display

  1. The Display Assignment Grades checkbox is checked by default
  2. The Display Course Grade checkbox is unchecked by default

2. Grade Conversion

  1. Grade Type dropdown choices: Letters grade +/- (default choice), Simple letter grade, Pass/Fail. Grade types cannot be defined at the level of the individual student. Depending on which type is selected, all students in a class will see either a letter grade or a P/NP mark, if the course grade is released to them.
  2. The Change grade type button: Refreshes the page with the percent table matching the scheme chosen in the dropdown. Other changes to the page are retained, even if they have not yet been saved.
  3. If course grades have been manually entered and the instructor attempts to change the grade type, upon save, the application will throw an error message: "You have manually entered course grades of the type X. To change the course grade type, you must first undo all manually overwritten course grades."
  4. Reset to default values link refreshes the page. The minimum % values are returned to the institution default. Grade type remains the same.

3. Grade/percent table

  1. If the values are changed, then the instructor changes to a different scheme, then switches back, the GB remembers the instructors previously entered scheme.
  2. Manually overwritten course grades are unaffected by changes to the Grade/percent table. If there are manually entered course grades, the application throws a warning message: "You have changed the grading scheme. All manually overwritten course grades remain unaffected."
  3. Minimum Percent text box validation: Values must be >= 0. Decimal values are accepted with no practical limitation on the number of places. Commas and other non-numerical characters, including % are not accepted. Their inclusion will throw an error message: "Please enter numbers only in the Minimum Percent Value field. The percent value must be > 0."

4. Save button

  1. Upon successful validation, returns the user to the overview, with a confirmation message saying "Student Feedback Options Saved"
  2. The save button is the only mechanism for saving feedback data. The enter key executes Save if the check boxes or text entry fieds have focus. If the Grade Type selector has been activated, the enter key will not execute the save.
  3. If newly entered data is not saved, it will be discarded upon using any of the gradebook's internal (as opposed to the browser's or Sakai framework's) navigation or find/show functionality. The gradebook supports the back button, although this is not currently true of most Sakai tools.

5. Cancel button

  1. Returns the user to the overview, with no confirmation message.

G. Remove assignment -- View 6

Wireframe of View 6

1. Remove this assignment checkbox

  1. Default value: unchecked
  2. Validation: The form can't submit unless this button is checked.

2. Remove button

  1. Marks the assignment as 'removed' in the database. It no longer appears in the database, but in an emergency could be recovered by an administrator. The tool then returns view 1 (overview) with a confirmation message : "Assignment deleted"
  2. If the checkbox above has not been selected when submit is pressed, a validation message should appear: "You must confirm this deletion by checking the box below before removing."
  3. The enter key executes Remove if the check box has focus

3. Cancel

  1. The user is returned to the assignment details page.

H. Student View -- View 7

Wireframe of Student View

1. Cumulative Score

  1. Visibility: This is visible if the instructor has checked the 'Display Assignment Grades' checkbox in view 5.
  2. Calculation: The points taken equals the total of available points for assignments the instructional staff has scored for this assignment.

2. Course Grade

  1. Visibility: This is visible if the instructor has checked the 'Display Final Grade' checkbox in view 5.
  2. Calculation: The final grade display is always based on the percentage of points scored out of the total possible points of all assignments entered in the gradebook. If the instructor releases the course grade before all of the assignments have been scored, the grades that the students see at any one time are unlikely to reflect their current performance in the class.

VI. Functional Roadmap

The following is a partial list of features that will not be included in the 2.0 gradebook, but which are expected to be prioritized for post 2.0 development:

  1. A richer set of grading schemes, including letter grades, percentages (applied to assignments and categories of assignments), automatic dropping of lowest score(s), and class curves
  2. An expanded set of instructional roles
  3. Assignment of instructional staff and students to course subgroups
  4. Statistical analyses of student performance
  5. Grader commenting
  6. Detailed grading histories
  7. A detailed view and grading of individual students
  8. Fuller interoperability with other Sakai tools
  9. Basic interface for course grade submission
