I. Introduction

The Gradebook is a tool that instructors can use to record, calculate, and disseminate grade information online. The principal aim of this development effort has been to produce a simple, open source grade management utility that can be deployed as a stand-alone web servlet or as a tool within Sakai.  A secondary aim of this effort has been to provide for the Sakai-deployed Gradebook a service that other Sakai tools can use to store and manage assignment scores. Design and development of the Gradebook has been led by UC Berkeley. Plans for further development of the Gradebook are outlined in Section VII.

II. Summary

A. The Sakai 2.3 Gradebook enables instructors to:

  1. Auto calculate course grades
  2. Establish and edit a mapping of the course letter grade to a 100 percent scale
  3. Add, view, edit and release to students the class assignments/assessments (straight point values only)
  4. Manually enter, view, edit and release to students their scores and grades
  5. Transmit scores to the Gradebook from tools such as Tests & Quizzes and Assignments, thereby creating a corresponding assignment in the Gradebook and recording student scores for that assignment.
  6. Export scores and grades in excel, csv format
  7. Upload CSV files containing columns of scores and then import these as Gradebook assignments
  8. Filter student lists by section
  9. Keep a log of grading activity for each student's work

B. The Sakai 2.3 Gradebook enables each student to:

  1. View his/her own scores and grades, if they have been released by the instructor(s).

III. Functional Requirements

Sections A through P correspond to all available views in the Sakai 2.3 Gradebook. All actions described in sections B through O are available only to the Instructor role.

A. Gradebook Menu

  1. Link to inactive views. The Overview is the entry page.

B. Overview

  1. Sort display table by assignment name, due date, released setting, class average and total points possible
  2. View class average scores for each assignment
  3. Access the assignment details views
  4. Access the course grade view

C. Assignment Detail

  1. View total possible points for an assignment and the class average
  2. Page through all assignment summaries one by one
  3. Filter the grading table by student name or id.
  4. Set the number of students to view per page of the grading table
  5. Page through all students based on the number of students to view per page
  6. Access the assignment settings page
  7. Remove assignments
  8. Sort the grading table by student name, student ID and score
  9. Enter, save and edit assignment scores
  10. Filter student list by section
  11. Display log of assignment grade changes for each student

D. Course Grade

  1. View total possible points for all assignments and the cumulative class average
  2. Filter the grading table by student name or id.
  3. Set the number of students to view per page of the grading table
  4. Override auto calculated course grade
  5. Sort the grading table by student name, student ID, cumulative assignment point totals per student, and calculated course grade (with corresponding percentages).
  6. Export student IDs, student names, and course grades
  7. Filter student list by section
  8. Display log of course grade changes (if any) for each student

E. Roster

  1. Filter the grading table by student name or id.
  2. Set the number of students to view per page of the grading table
  3. Page through all students based on the number of students to view per page
  4. View all assignments, each student's scores on all assignments, and the cumulative assignment point totals for each student.
  5. Export student name, student ID, and scores for all assignments in .csv and .xls
  6. Filter student list by section

F. Add Assignment

  1. Enter an assignment name, a required step
  2. Enter an assignment point value, a required step
  3. Select an assignment due date, an optional step
  4. Choose whether to count assignment towards final grade, an optional step
  5. Change an assignment’s release status, an optional step

G. Edit Assignment

  1. Change an assignment name
  2. Change an assignment point value
  3. Change an assignment due date
  4. Choose whether to count assignment towards final grade
  5. Change an assignment’s release status

H. Remove assignment

  1. Select remove option

I. Grade Options

  1. Choose to display released assignment scores to students
  2. Choose to display cumulative score, course grades and calculated assignment grades to students
  3. Select the grade type for the course grade and assignment level grades in the student view: letter grades, letter grades + - (default), pass/fail
  4. Define percent ranges for letter grades or the pass/fail threshold

J. Upload/Import

  1. Upload a spreadsheet (csv format) to the loading dock
  2. Import assignments from a spreadsheet that is already in the loading dock
  3. Delete a spreadsheet from the loading dock
  4. Sort uploaded spreadsheets in the loading dock by Title, Creator, or Date

K. Upload a spreadsheet (csv) format to Loading Dock

  1. Enter a title for the file as it will appear in the loading dock
  2. Select a local file (via file picker) for upload to loading dock
  3. Upload file via Save button
  4. Cancel upload via Cancel button

L. Verify Upload 

  1. View spreadsheet contents in table display matching original column and row layout
  2. Cancel upload via Cancel button

M. Import Preview   

  1. View  spreadsheet contents with radio button selectors in column headings
  2. Check radio button for the assignment to be imported
  3. Select Import Selected button to proceed with import
  4. Select Cancel to cancel import

N. Import Assignment and Scores   

  1. Enter an assignment name, a required step
  2. Enter an assignment point value, a required step
  3. Select an assignment due date, an optional step
  4. Choose whether to count assignment towards final grade, an optional step
  5. Change an assignment’s release status, an optional step

O. Delete Spreadsheet     

  1. Check “Remove this Spreadsheet from the Gradebook” option
  2. Select Remove button

P. Grade Report for Student

Each student will be able to:

  1. View his or her cumulative score for all assignments and corresponding percentage
  2. View his or her course grade, if released by instructor
  3. Sort assignments by title, due date, his or her score, total possible points per assignment, his or her grade for each assignment

IV. Roles and Groups

The Sakai site and section tools will enable authorized users to:

  1. Add other instructors. The Gradebook will recognize only a single instructor role. Once added to a site, all instructors will be fully authorized to view, create and edit in all areas of a Gradebook.
  2. Add a Teaching Assistant. The Gradebook will recognize only a single TA role. Once added to the site and assigned one or more sections to manage, the TA will be authorized to enter grades exclusively for the students in those assigned sections.
  3. Add students. The Gradebook will recognize only a single student role.

7. Save

  1. The Save button saves newly entered data and refreshes the view (average score is updated and 'changes saved' message added to top of page). The last executed search and students per page values are retained and displayed upon refresh.
  2. The save button is the only mechanism for saving score data. The enter key executes Save if the score boxes have focus.
  3. If newly entered data is not saved, it will be discarded upon using any of the Gradebook's internal (as opposed to the browser's or Sakai framework's) navigation or find/show functionality. The Gradebook supports the back button, although this is not currently true of most Sakai tools.
  4. If two or more users are simultaneously attempting to edit the same student's score, the second of the scores to be submitted will overwrite the first.
  5. If bad score values are entered, upon save an error message is generated

8. Cancel button

  1. Any new form data is discarded and the user returns to the Overview page.

9. Point Entry

  1. Values must be >= 0. Decimal values of up to 2 places are accepted, e.g. 9.75. Commas and other non-numerical characters are not accepted.
  2. Illegitimate values generate an error message: "Please enter only scores >=0 of up to 2 decimal places. "
  3. Error messages are also generated next to each of the fields with bad values: "Please enter valid grade."
  4. After values are saved, the maximum number of visible characters in text box = 3; these are right aligned. Values > 3 characters are scrollable.
  5. Scores submitted via the Tests and Quizzes tool cannot be edited in the Gradebook.

10. Grade Change Log

  1. Clicking the log icon reveals the pop-up window shown in image above.
  2. Every grade change made, including the initial grade set, is listed in reverse chronological order.
  3. The date, score set, and person or agent making the change is shown
  4. A small "[x]" in the corner of the pop-window can be clicked to close it.

D. Course Grade -- View 4

Screenshot of View 4

1. Course Summary

  1. Points: Total possible points of all assignments currently entered into the Gradebook directly and/or via SAMigo.
  2. Average Score: The mean of all possible score values for all assignments. Blank scores are treated as zeros for calculation purposes.

2. Calculated Grade

  1. The calculated grade field is by default filled with the grade calculated by the Gradebook (A, B . . .or A-, B+ . . . or P/NP) and the percentage the grade is based on.
  2. The grade is always calculated as the current percentage of points scored out of the total possible points of all assignments entered in the Gradebook. Consequently, the grades that the instructors see are unlikely to reflect students' current performance in the class, unless or until all of the assignments have been scored.
  3. For purposes of calculating the course grade, a student's cumulative average percentage is never rounded up. For example, a student who achieves an overall percentage of 79.99 would receive a course grade of C+, if the lower threshold for a B- is 80%.

3. Grade Override

  1. Override values must be of the appropriate grade type (A, B . . . or A-, B+ . . .or P/NP) or an error is generated: "The Gradebook uses the grade type [pass/not pass|simple letter grade|letter grade +/-]. Please enter only values that correspond to the grade type X."
  2. Error messages are also generated next to each of the fields with bad values: "Please enter valid grade."
  3. Saved overrides appear in the entry field as editable values.
  4. Values identical to the calculated course grade are treated as legitimate entries and do not generate an error.
  5. Deleting a saved overridden value, leaving the entry field blank and executing a save returns the course grade to the calculated value.

4. Export

  1. Files are exported in .xls and .csv format
  2. The file name format is "course_grade_[courseID]__date.*". If the class number is not available use 'course_grade_050611. Date format should follow the style guide.
  3. Exported fields and the names of the Excel column headers are Student Name, Student ID, Course Grade. The full student display names and complete assignment names are exported. Point values are chopped at 2 decimal places.
  4. Scores are listed in the assignment name column
  5. Point values are chopped at 2 decimal places.
  6.    Exported data is for the course grade only
  7. All available roster data is exported. Filtering the view by student does not affect the output.
  8. The exported data is always sorted by the application default, student display name
  9.    There is no navigation state change after clicking these buttons

I. Grade Options -- View 9

Screenshot of View 14

1. Grade Display

  1. The released assignment grades display checkbox is checked by default
  2. The cumulative score, course grades and calculated assignment grades display checkbox is unchecked by default

2. Grade Conversion

  1. Grade Type dropdown choices: Letters grade +/- (default choice), Simple letter grade, Pass/Fail. Grade types cannot be defined at the level of the individual student. Depending on which type is selected, all students in a class will see either a letter grade or a P/NP mark, if the course grade is released to them.
  2. The Change grade type button: Refreshes the page with the percent table matching the scheme chosen in the dropdown. Other changes to the page are retained, even if they have not yet been saved.
  3.   4. Reset to default values link refreshes the page. The minimum % values are returned to the institution default. Grade type remains the same.

3. Grade/percent table

  1. If the values are changed, then the instructor changes to a different scheme, then switches back, the GB remembers the instructors previously entered scheme.
  2. Manually overwritten course grades are unaffected by changes to the Grade/percent table. If there are manually entered course grades, the application throws a warning message: "You have changed the grading scheme. All manually overwritten course grades remain unaffected."
  3. Minimum Percent text box validation: Values must be >= 0. Decimal values are accepted with no practical limitation on the number of places. Commas and other non-numerical characters, including % are not accepted. Their inclusion will throw an error message: "Please enter numbers only in the Minimum Percent Value field. The percent value must be > 0."

4. Save button

  1. Upon successful validation, returns the user to the overview, with a confirmation message saying "Student Feedback Options Saved"
  2. The save button is the only mechanism for saving feedback data. The enter key executes Save if the check boxes or text entry fields have focus. If the Grade Type selector has been activated, the enter key will not execute the save.
  3. If newly entered data is not saved, it will be discarded upon using any of the Gradebook's internal (as opposed to the browser's or Sakai framework's) navigation or find/show functionality. The Gradebook supports the back button, although this is not currently true of most Sakai tools.

VI. Functional Roadmap

The following is a partial list of features that will not be included in the 2.0 Gradebook, but which are expected to be prioritized for post 2.0 development:

  1. A richer set of grading schemes, including letter grades, percentages (applied to assignments and categories of assignments), automatic dropping of lowest score(s), and class curves
  2. An expanded set of instructional roles
  3. Assignment of instructional staff and students to course subgroups
  4. Statistical analyses of student performance
  5. Detailed grading histories
  6. A detailed view and grading of individual students
  7. Basic interface for course grade submission
